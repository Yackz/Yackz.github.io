CSS Specificity

การเขียน css มักจะมีการประกาศ tag ซ้ำๆกัน และมีหลายไฟล์มากๆ ดังนี้เราจึงต้องนำ CSS Specificity มาเพื่อหาว่า browser จะนำ tag ไหนที่มีค่ามากที่สุดแล้วนำมาประมวลผลให้แก่ผู้ใช้

แล้วเราจะรู้ได้ไงล่ะ พวกโค้ดต่างๆที่เราเขียนมานั้น คำสั่งไหน มีค่าเท่าไหร่ หรือจะประกาศตัวแปรอย่างไร ให้ตัวแปรนั้นถูกนำมาใช้ได้อย่างถูกต้อง ก่อนอื่นเลย เราต้องรู้ลำดับความสำคัญของแต่ละแบบ หรือเรียกง่ายๆว่า ค่าพลังของการเขียนแต่ละแบบก่อน โดยทางเราได้ทำสรุปไว้ให้แล้ว ทางซ้ายมือในหัวข้อ ลำดับความสำคัญ

ตัวอย่าง การคำนวณและรูปแบบของลำดับแต่ละอัน

1. Inline style หรือการประกาศ style ไว้ในไฟล์ html ด้านหลังของ tag นั้นๆ
เช่น tag p style = "color = red"; จากตัวอย่างนี้จะมีค่าเท่ากับ 1000
2. การประกาศ ID ใน tag ของ html
เช่น #kong {} จากตัวอย่างนี้จะมีค่าเท่ากับ 100
3. การเขียน CSS โดยการเรียกจาก Class
เช่น .sidebar{} จากตัวอย่างนี้จะมีค่าเท่ากับ 10
4. การประกาศจาก tag ของ html โดยตรง
เช่น h3{} จากตัวอย่างนี้จะมีค่าเท่ากับ 1

จากที่เราเรียนรู้ค่าของการประกาศแบบต่างๆแล้ว เรามาดูการใช้งานจริงดีกว่า

Example

แบบที่ 1
ul li.active a{ color : gold; }
แบบที่ 2
ul li a{ color : red; }
ผลรวมของแบบที่ 1 = tag + tag + class + tag = 1 + 1 + 10 + 1 = 13
ผลรวมของแบบที่ 2 = tag + tag + tag = 1 + 1 + 1 = 3
ผลรวมของแบบที่ 1 มากกว่า แบบที่ 2 ดังนั้น browser จะแสดงผลสีทอง ตามแบบที่ 1

หมายเหตุ :

ถ้าเกิดเราคำนวณมาแล้วคะแนนมีค่าเท่ากัน เราต้องย้ายไปดูหน้า html ว่ามีการเชื่อม style อันไหนเป็นอันสุดท้าย style ที่ถูกเชื่อมเป็นตัวสุดท้าย ตัวนั้นจะเป็นตัวกำหนด CSS ให้แก่ browser เราเรียกวิธีนี้ว่า CSS Cascade

ลำดับความสำคัญ

Inline style มีค่า 1000
ID มีค่า 100
Class มีค่า 10
พวก Tag ต่างๆ มีค่า 1
